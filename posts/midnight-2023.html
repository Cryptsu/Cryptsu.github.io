Idea

Well, this is just classic RSA encryption. On its own, this scheme is definitely not breakable. However, the author provides some additional values h0,h1h_0, h_1h0​,h1​ so that somehow, we can use them to decrypt the encrypted value ccc:

h0=p+qb2h1=q+pa2h_0 = p + qb^2 \\ h_1 = q + pa^2h0​=p+qb2h1​=q+pa2

where aaa, bbb is some random value ∈[0,21024)\in [0, 2^{1024})∈[0,21024).

My teammate immediately recognized (god 😢) that:

h0h1n is a close approximation to ab\sqrt{\frac{h_0h_1}{n}} \text{ is a close approximation to } {ab}nh0​h1​​​ is a close approximation to ab

For which you can reason it by doing the following calculations:

h0h1n=(p+qb2)(q+pa2)pq=p+qb2qq+pa2p=(pq+b2)(qp+a2)=a2b2+1+b2qp+a2pq=a2b2+1+abbaqp+ababpq=a2b2+1+ab(baqp+abpq)\begin{aligned} \frac{h_0h_1}{n} & = \frac{(p + qb^2)(q + pa^2)}{pq} \\ & = \frac{p + qb^2}{q} \frac{q + pa^2}{p} \\ & = (\frac{p}{q} + b^2)(\frac{q}{p} + a^2) \\ & = a^2b^2 + 1 + b^2\frac{q}{p} + a^2\frac{p}{q} \\ & = a^2b^2 + 1 + ab\frac{b}{a}\frac{q}{p} + ab\frac{a}{b}\frac{p}{q} \\ & = a^2b^2 + 1 + ab(\frac{b}{a}\frac{q}{p} + \frac{a}{b}\frac{p}{q}) \end{aligned} nh0​h1​​​=pq(p+qb2)(q+pa2)​=qp+qb2​pq+pa2​=(qp​+b2)(pq​+a2)=a2b2+1+b2pq​+a2qp​=a2b2+1+abab​pq​+abba​qp​=a2b2+1+ab(ab​pq​+ba​qp​)​

Let's set baqp+abpq\frac{b}{a}\frac{q}{p} + \frac{a}{b}\frac{p}{q}ab​pq​+ba​qp​ to be kkk.

a2b2+1+kab a^2b^2 + 1 + kaba2b2+1+kab

If kkk is small enough to aaa and bbb, we can approximate it to be

h0h1n≈a2b2+kab+(k2)2≈(ab+k2)2\begin{aligned} \frac{h_0h_1}{n} & \approx a^2b^2 + kab + (\frac{k}{2})^2 \\ & \approx (ab + \frac{k}{2})^2 \end{aligned}nh0​h1​​​≈a2b2+kab+(2k​)2≈(ab+2k​)2​

Since qp,ba\frac{q}{p}, \frac{b}{a}pq​,ab​ and their reciprocals involve in the sum making up kkk, the bigness of kkk depends on max(p,q)min(p,q)\frac{max(p, q)}{min(p, q)}min(p,q)max(p,q)​ and max(a,b)min(a,b)\frac{max(a, b)}{min(a, b)}min(a,b)max(a,b)​. And you can reason that for any two numbers x,yx, yx,y generated at random in the same interval, the ratio max(x,y)min(x,y)\frac{max(x, y)}{min(x, y)}min(x,y)max(x,y)​ is expected to be small.


During local test, I realized that kkk is very small (like, 222 or 333 or something...). In fact, let's plot kkk for different values of p,q,a,bp, q, a, bp,q,a,b and see how it goes 🙂

The left column represents the percentage of occurances for a certain kkk. And you can see for ALMOST ALL cases, kkk will just land around on some value ≤6\le 6≤6. (yes, while you can argue that the spike of 222 is just 5%, so the rest could be much bigger, however, the probability of kkk landing >20> 20>20 is just 12%)

Also, you can notice that the plot is flat before a sudden spike around k=2k = 2k=2. This happens because kkk can be represented in the form of 1x+x\frac{1}{x} + xx1​+x, and by Cauchy inequality, 1x+x≥2\frac{1}{x} + x \ge 2x1​+x≥2 for any x>0x > 0x>0.


Anyways, this means in most cases, we can expect:

h0h1n≤(ab+3)2h0h1n≤ab+3 \frac{h_0h_1}{n} \le (ab + 3)^2 \\ \sqrt{\frac{h_0h_1}{n}} \le ab + 3 \\nh0​h1​​≤(ab+3)2nh0​h1​​​≤ab+3

or if you are really skeptical, you can bound it to be ≤ab+10000\le ab + 10000≤ab+10000, just to be sure 🙂 This means that we can find ababab by bruteforcing values around h0h1n\sqrt{\frac{h_0h_1}{n}}nh0​h1​​​.

Solve

Knowing ababab, we now have 4 equations with 4 variables to solve:

pq=nab=(whatever ab is)h0=p+qb2h1=q+pa2 pq = n \\ ab = \text{(whatever } ab \text{ is)} \\ h_0 = p + qb^2 \\ h_1 = q + pa^2pq=nab=(whatever ab is)h0​=p+qb2h1​=q+pa2

for which we can lend the power of magic Sagemath to solve for p,qp,qp,q.

    I = H.ideal()
    if I.dimension() == -1:

    elif I.dimension() == 0:
